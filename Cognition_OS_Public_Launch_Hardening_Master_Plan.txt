Cognition OS
Public Launch Hardening Master Plan
Final-product deployment command document for the current uploaded codebase

Mission
Turn the current Cognition OS codebase into a public, paid, deployment-ready product that can acquire users, charge them, onboard them, deliver the core learning OS experience, monitor itself, and fail safely while the founder is unavailable for active testing.
Operating constraint
The founder will not be available for active product testing for 10-15 days. Therefore every module below must be implemented with automated verification, deterministic scripts, clear acceptance criteria, and production-grade observability. Do not rely on founder walkthroughs as a quality gate.

How this document must be used
Treat this as a boss-to-engineering execution order. The team should not reinterpret the target as MVP. The target is public deployment readiness.
Treat the current repository as the base. Do not rewrite the whole app unless a module explicitly says a subsystem must be consolidated.
Do not add new visible features until install, build, billing, onboarding, security, and learning-loop correctness are stable.
Every module must end with automated tests. Manual clicking is allowed only as a final smoke check, not as the proof of correctness.
Every visible user-facing feature must be either fully working, clearly gated by plan, admin-only, or hidden. No orphaned routes or broken claims.
Every expensive feature must have rate limits, plan limits, abuse limits, cost monitoring, and graceful upgrade messaging.
Every mutating route must be idempotent where browser retries, double-clicks, payment webhooks, or worker retries are possible.
